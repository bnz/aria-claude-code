# Промпт 20: CMS — Редактор Translations

## Контекст
Draft система готова (промпт 19). Создаём первый раздел CMS — редактор UI-переводов.

## Задача
Создай редактор для раздела Translations в CMS:

1. **Страница** `Translations Editor`:
   - Загрузка `translations.{en,lv,ru}.json` из GitHub (через contentManager)
   - Проверка наличия draft в localStorage (через draftManager)

2. **Интерфейс**:
   - Табы/секции по группам: `header`, `footer`, `buttons`
   - Для каждого ключа — 3 поля ввода рядом (en | lv | ru) или табы по языкам
   - Возможность добавить новый ключ (ключ + значения на 3 языках)
   - Возможность удалить ключ (из всех языков сразу)

3. **Валидация**: Zod (TranslationsSchema) при редактировании
4. **Автосохранение**: каждое изменение → draft в localStorage
5. **Индикатор изменений**: badge «Изменено» рядом с вкладкой Translations

## Требования к тестам
- Редактор загружает и отображает все ключи
- Добавление/удаление ключа работает синхронно для всех языков
- Изменения сохраняются в draft
- Zod-валидация срабатывает при некорректных данных
- Запусти тесты

## Ожидаемый результат
Функциональный редактор UI-переводов с валидацией и автосохранением.
