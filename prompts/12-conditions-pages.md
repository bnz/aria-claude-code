# Промпт 12: Страницы состояний/симптомов (/conditions) — SEO core

## Контекст
Статьи готовы (промпт 11). Создаём ключевой для SEO раздел — conditions.

## Задача
Создай страницы для раздела conditions (это **основной SEO-рост** проекта):

1. **Список состояний** `app/[lang]/conditions/page.tsx`:
   - Загрузка `conditions/index.json` + данные каждого условия
   - Карточки: заголовок, intro (обрезанный)
   - Только опубликованные, сортировка по `order`
   - Ссылки на `/[lang]/conditions/[slug]`

2. **Страница состояния** `app/[lang]/conditions/[slug]/page.tsx`:
   - `generateStaticParams` — все slug × все языки
   - Заголовок + intro
   - Секции (text, bullets, image) через `ContentSections`
   - Противопоказания (contraindications) — блок-предупреждение
   - FAQ секция — аккордеон или список вопросов-ответов
   - CTA блок: «Запишитесь на консультацию» + телефон

3. Mobile-first, семантическая HTML разметка (h1, h2, article, section)

## Требования к тестам
- Список показывает только published
- FAQ рендерится как вопрос-ответ
- Противопоказания отображаются
- CTA содержит ссылку `tel:`
- generateStaticParams корректен
- Запусти тесты и `next build`

SEO-оптимизированные страницы условий с FAQ, противопоказаниями и CTA.
