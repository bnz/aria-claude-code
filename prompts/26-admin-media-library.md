# Промпт 26: CMS — Media Library (управление изображениями)

## Контекст
Все редакторы созданы (промпты 20-25). Некоторые из них используют кнопку «Выбрать изображение» — теперь реализуем Media Library.

## Задача
Создай модальную Media Library для CMS:

1. **Модальное окно** `MediaLibraryModal`:
   - Открывается при клике на кнопку выбора изображения в любом редакторе
   - Отображает все изображения из `/public/media/` (загрузка через GitHub API — listFiles)
   - Сетка миниатюр
   - Выбор изображения → возвращает путь (`/media/filename.jpg`)
   - Закрытие по крестику/escape/клику вне модали

2. **Загрузка нового изображения**:
   - Кнопка «Upload» в модальном окне
   - Выбор файла (input type=file, accept images)
   - Предпросмотр перед загрузкой
   - Загрузка = **отдельный commit** через GitHub API (commitFile → `/public/media/filename`)
   - После загрузки: обновить список, новое изображение доступно для выбора
   - Показать прогресс загрузки

3. **Важно**: загрузка изображения запускает deploy (GitHub Actions). Показать уведомление: «Изображение загружено. Deploy запущен.»

4. **ImagePicker компонент**: обёртка для использования в формах
   - Показывает текущее выбранное изображение (превью)
   - Кнопка «Выбрать» → открывает MediaLibraryModal
   - Кнопка «Убрать» → очистить поле
