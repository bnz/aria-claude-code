# Промпт 19: CMS — Draft система (localStorage)

## Контекст
GitHub API клиент готов (промпт 18). Нужна система локальных черновиков.

## Задача
Создай систему draft/черновиков в localStorage:

1. **Draft Manager** (`src/lib/admin/draftManager.ts`):
   - `saveDraft(key, data)` — сохранить черновик
   - `loadDraft(key)` → данные или null
   - `hasDraft(key)` → boolean
   - `clearDraft(key)` — удалить черновик
   - `clearAllDrafts()` — очистить всё
   - `getDirtyKeys()` — список изменённых ключей (diff vs загруженная версия)

2. **Интеграция с редакторами**:
   - При загрузке данных из GitHub — проверять, есть ли локальный draft
   - Если draft есть — предложить «Восстановить черновик» или «Сбросить к серверной версии»
   - При каждом изменении в форме — автосохранение в localStorage
   - Визуальный индикатор: «Есть несохранённые изменения» (жёлтая точка/badge)

3. **Diff tracking**:
   - Хранить оригинальную версию (с GitHub) и текущую (draft)
   - Метод `getChangedFiles()` — возвращает список файлов с изменениями (для Publish)

4. Draft разрешён даже во время деплоя (коммиты заблокированы, но редактирование — нет)

## Требования к тестам
- saveDraft/loadDraft работают корректно
- getDirtyKeys возвращает только изменённые файлы
- Восстановление черновика после перезагрузки
- Запусти тесты

## Ожидаемый результат
Надёжная система черновиков с автосохранением и отслеживанием изменений.
