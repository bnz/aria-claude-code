# Промпт 16: CMS — GitHub OAuth авторизация

## Контекст
Публичный сайт завершён (промпты 01-15). Начинаем CMS. Первый шаг — авторизация.

## Задача
Создай авторизацию для CMS через GitHub OAuth:

1. **Роут `/admin`** — точка входа CMS:
   - Динамический import (код CMS не грузится на публичных страницах)
   - `app/admin/layout.tsx` и `app/admin/page.tsx`

2. **Экран логина** (`AdminLogin`):
   - Минимальный бандл: только кнопка «Sign in with GitHub»
   - GitHub OAuth flow (client-side, без backend):
     - Вариант 1: GitHub OAuth App + device flow
     - Вариант 2: Personal Access Token (запасной вариант, простой input)
   - Реши, какой подход использовать, зафиксируй в DECISIONS.md

3. **Auth контекст** (`AdminAuthContext`):
   - Хранение токена (sessionStorage, не localStorage — безопаснее)
   - Проверка доступа к репозиторию
   - Logout

4. **Защита роутов**: если не авторизован — показывать только экран логина
5. После входа — подгружать основной бандл CMS

- Неавторизованный пользователь видит только экран логина
- После авторизации доступна CMS
- Logout очищает токен
- Код CMS не попадает в бандл публичного сайта (проверь chunk names)
- Запусти тесты
