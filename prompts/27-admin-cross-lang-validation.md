# Промпт 27: CMS — Cross-language валидация и предупреждения

## Контекст
Все редакторы и Media Library готовы (промпты 20-26). Добавляем проверки согласованности между языками.

## Задача
Реализуй cross-language валидацию согласно секции 9 спека:

1. **Cross-language валидатор** (`src/lib/admin/crossLangValidation.ts`):
   - Проверка: `id` и `slug` совпадают между en/lv/ru
   - Проверка: если сущность есть в одном языке — должна быть во всех
   - Проверка: обязательные поля заполнены во всех языках
   - Проверка: SEO поля (title, description) есть на всех языках
   - Для articles/conditions: количество секций совпадает между языками

2. **Предупреждения в UI**:
   - Если пользователь изменил только один язык: «Вы изменили только RU. EN и LV не обновлены. Продолжить?»
   - Если обязательное поле пустое в каком-то языке: конкретное сообщение с указанием языка и поля
   - Предупреждения показываются:
     a) В реальном времени — иконка ⚠️ рядом с пустыми полями
     b) Перед Publish — полный список проблем

3. **Режимы** (настройка в CMS):
   - Строгий: блокировать Publish при неполных языках
   - Мягкий: разрешить Publish после подтверждения
   - По умолчанию: мягкий (зафиксируй в DECISIONS.md)

## Требования к тестам
- Несовпадение id/slug между языками → ошибка
- Пустое обязательное поле в одном языке → предупреждение
- Изменение только одного языка → предупреждение
- Строгий режим блокирует Publish
- Мягкий режим показывает подтверждение
- Запусти тесты
